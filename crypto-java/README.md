Java REST API сервис для криптографических операций с документами: подписание, верификация, шифрование, дешифрование и хеширование.

## Функциональность

| Метод  | Endpoint                 | Описание                                             |
| ------ | ------------------------ | ---------------------------------------------------- |
| POST   | `/crypto/sign`           | Подписание документа (PKCS#7, `detached=true/false`) |
| POST   | `/crypto/verify`         | Верификация подписи                                  |
| POST   | `/crypto/encrypt`        | Шифрование документа                                 |
| POST   | `/crypto/decrypt`        | Дешифрование документа                               |
| POST   | `/crypto/hash`           | Вычисление SHA-256 хеша                              |
| DELETE | `/crypto/files?name=...` | Удаление документа                                   |
| GET    | `/crypto/operations`     | Лог операций                                         |

Все POST-запросы принимают файл через `multipart/form-data` (поле `file`, заполняется в body в Postman).
Коллекция postman для запросов приложена в ../demo_api_requests.postman_collection.json

## Запуск

На данный момент приложение запускается с помощью maven-execute-plugin
Проверьте что путь до mvn установлен в PATH или исправьте скрипт run.sh в соответствии с своей системой
Предполагается, что вы уже сгенерировали сертификаты с помощью скрипта generate_certs.sh.

### 1. Настройка конфигурации

Заполните переменные в `.env` при необходимости (на текущий момент)

### 2. Запуск (Linux / Mac)

```bash
./run.sh
```

## Критические проблемы на текущий момент (требуется доработка)

Проблема с повторным использованием файлов с одинаковыми именами, поскольку имя было указано как UNIQUE - требует доработки.

Для sign, encrypt и hash - требуется чтобы имя файла не значилось в базе в сохраненных.
Как решение - удалять по имени файла после проверки операций либо использовать файлы с разными именами.

Операции работы с базой на текущий момент не поддерживают транзакционность.

## Конфигурация (.env)

| Переменная    | Описание                |                     По умолчанию                      |
| ------------- | ----------------------- | :---------------------------------------------------: |
| `PORT`        | Порт HTTP               |                        `8081`                         |
| `DB_URL`      | JDBC URL (H2)           | `jdbc:h2:file:./data/cryptoservices;AUTO_SERVER=TRUE` |
| `DB_USER`     | Пользователь БД         |                         `sa`                          |
| `DB_PASS`     | Пароль БД               |                         `sa`                          |
| `KS_PATH`     | Путь к PKCS#12 keystоre |                `../certs/keystore.p12`                |
| `KS_PASSWORD` | Пароль keystore         |                      `changeit`                       |
| `KS_ALIAS`    | Алиас сертификата       |                       `server`                        |
