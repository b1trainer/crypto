REST API сервис для криптографических операций с документами

Проект представлен в двух реализациях:

- **crypto-java/** — Java 17 + Maven (Embedded Tomcat, Jersey, Bouncy Castle)
- **crypto-go/** — Go 1.26+ (Standard Library, h2go, pkcs7)

## Предназначение

Сервис предназначен для выполнения криптографических операций над произвольными файлами через HTTP REST API:

| Операция         | Описание                                                                                     |
| ---------------- | -------------------------------------------------------------------------------------------- |
| **Подписание**   | Создание PKCS#7 подписи (attached/detached) с использованием сертификата из PKCS#12 keystore |
| **Верификация**  | Проверка подлинности PKCS#7 подписи документа                                                |
| **Шифрование**   | Симметричное/асимметричное шифрование файла                                                  |
| **Дешифрование** | Восстановление исходного файла из зашифрованных данных                                       |
| **Хеширование**  | Вычисление SHA-256 хеша файла                                                                |
| **Управление**   | Удаление документов, просмотр лога операций                                                  |

Сервис хранит исходные и обработанные документы в in memory базе данных, и логирует операции в отдельную таблицу

## API Endpoints

### Java-сервис

| Метод  | Путь                     | Описание                                     |
| ------ | ------------------------ | -------------------------------------------- |
| POST   | `/v1/crypto/sign`        | Подписание документа (`detached=true/false`) |
| POST   | `/v1/crypto/verify`      | Верификация подписи                          |
| POST   | `/v1/crypto/encrypt`     | Шифрование документа                         |
| POST   | `/v1/crypto/decrypt`     | Дешифрование документа                       |
| POST   | `/v1/crypto/hash`        | Вычисление SHA-256 хеша                      |
| DELETE | `/v1/crypto/files?name=` | Удаление документа по имени                  |
| GET    | `/v1/crypto/operations`  | Лог операций                                 |

### Go-сервис

| Метод | Путь                    | Описание                                     |
| ----- | ----------------------- | -------------------------------------------- |
| POST  | `/v2/crypto/sign`       | Подписание документа (`detached=true/false`) |
| POST  | `/v2/crypto/verify`     | Верификация подписи                          |
| POST  | `/v2/crypto/encrypt`    | Шифрование документа                         |
| POST  | `/v2/crypto/decrypt`    | Дешифрование документа                       |
| POST  | `/v2/crypto/hash`       | Вычисление SHA-256 хеша                      |
| GET   | `/v2/crypto/operations` | Лог операций                                 |

> **Примечание:** Все POST-запросы используют `multipart/form-data` с полем `file`.

## Запуск

### Генерация сертификатов

Создайте директорию certs в корне проекта. 
Для генерации сертфиикатов используйте скрипт `generate_certs.sh`:

```bash
./generate_certs.sh
```

> **Важно:** В скрипте генерации на текущий момент жёстко задан путь `JAVA_HOME` — исправьте его под вашу систему.

### Запуск Java-сервиса

```bash
cd crypto-java
./run.sh
```

### Запуск Go-сервиса

```bash
cd crypto-go
go run main.go
```

## Известные проблемы

- Имя файла указано как `UNIQUE` в БД, что вызывает ошибку при повторной загрузке файла с тем же именем для операций `sign`, `encrypt`, `hash`. Рабочее решение — использовать `DELETE /v1/crypto/files?name=...` перед повторной загрузкой файла с тем же именем (не реализовано для crypto-go на текущий момент)
- Операции с БД на текущий момент не обернуты в транзакции — при сбое на середине операции данные могут остаться в некорректном состоянии
- Поддержка TLS/mTLS ещё не реализована
- Мониторинг и observability требуют доработки
- Необходим переход на более стойкие схемы шифрования RSA-OAEP для go-модуля
- Необходим анализ и доработка с точки зрения безопасности приложения
